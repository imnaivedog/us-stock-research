# V5.7 Change Note

V5.7 is the implementation plan that turns the earlier ADR/core design into the current repo shape. It is not a separate ADR-number series; canonical ADR numbers remain in `core/adr.md`.

## Summary

V5.7 delivers the repo shape for:

- 3 main packages plus MCP: data, analytics, reports, MCP.
- YAML SoT for M/A pools and themes.
- M pool daily signal engine.
- A pool long-thesis timing engine with mcap anchors.
- 12 A pool signals.
- 35/30/35 A_Score.
- Read-only 4-tool MCP surface.
- LightOS/Postgres production cutover path.

## Key Deltas

| Area | V5.7 decision |
| --- | --- |
| A pool SoT | `config/a_pool.yaml`; DB mirrors only `pool`, `is_active`, `thesis_added_at` |
| M pool override | `config/m_pool_overrides.yaml` |
| Themes | `config/themes.yaml` plus `themes_master`, `themes_members`, `themes_score_daily` |
| Market-cap anchors | `thesis_stop_mcap_b` and `target_mcap_b`; price derived from `shares_outstanding` |
| shares outstanding | NULL means hold + `alert_log` WARN |
| Theme scores | Current-production/current-consumption; do not backfill just to make a signal |
| Verdict | LLM failures must fall back to deterministic `verdict_text` |
| Reports | Daily excludes simulated NAV/position section per ADR-033 |

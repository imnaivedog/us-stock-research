# V5.7 变更记录

V5.7 是把早期 ADR / core design 落到当前 repo 形态的 implementation plan。它不是一组新的 ADR 编号；canonical ADR 仍见 `core/adr.md`。

## 摘要

V5.7 交付 repo 形态：

- 3 个主 package + MCP：data、analytics、reports、MCP。
- M/A 池和 themes 以 yaml 为 SoT。
- M 池 daily signal engine。
- A 池 long-thesis timing engine，使用 mcap anchor。
- A 池 12 类信号。
- A_Score = 35/30/35 三维评分。
- read-only 4 工具 MCP 查询面。
- LightOS/Postgres production cutover 路径。

## 关键增量

| 范围 | V5.7 决策 |
| --- | --- |
| A 池 SoT | `config/a_pool.yaml`；DB 仅镜像 `pool`、`is_active`、`thesis_added_at` |
| M 池 override | `config/m_pool_overrides.yaml` |
| Themes | `config/themes.yaml` + `themes_master` / `themes_members` / `themes_score_daily` |
| 市值锚 | `thesis_stop_mcap_b` 和 `target_mcap_b`；价格由 `shares_outstanding` 反推 |
| shares_outstanding | NULL 时信号走 hold，并写 `alert_log` WARN |
| Theme scores | 现产现耗；不为了补信号而倒填历史 |
| Verdict | LLM 失败必须落 deterministic `verdict_text` |
| 输出 | 按 ADR-033，日报不含模拟 NAV / 仓位段 |

## V5+1 patch 进度

- P1 `p1-load-dotenv`：`usstock_data.db` 在 import 时加载 repo root `.env`，CLI 不再依赖 `source .env`；`load_dotenv` 保持 `override=False`，显式 shell env 优先。ADR-026。
- P2 `p2-normalize-postgres-url`：`DATABASE_URL` 支持 `postgres://` / `postgresql://` 输入，engine 创建前统一转为 `postgresql+psycopg://`，避免 SQLAlchemy 走 psycopg2 dialect。ADR-027。
- P3 `p3-ddl-three-phase-order`：`ddl.sql` 重排为 CREATE TABLE -> ALTER TABLE -> CREATE INDEX 三阶段，旧表会先补 V5.7 新列再建依赖索引。ADR-027。
- P4 `p4-etl-skip-noise`：`corporate_actions` / `fundamentals` 的 expected per-symbol skip 改为 DEBUG + INFO 汇总，`FMPTransientError` 仍保留 ERROR；已核查 polygon 域名引用，未在本 patch 切换 API provider。ADR-026。
- P5 `p5-ignore-backups`：`.gitignore` 增加 `*.bak.*` / `*.bak` / `*.bak.[0-9]*`，防临时备份文件进入 commit。ADR-026。
- P6 `p6-v5-plus-1-retro`：新增 `CHANGELOG.md` 与 `docs/changes/2026-04-V5_PLUS_1.md`，归档 V5+1 修补包背景、patch 对照、测试覆盖和后续边界。ADR-026。
- P2 follow-up `p2-db-url-password-fallback`：`DATABASE_URL` 缺 password 时从 `POSTGRES_PASSWORD` 补齐，修 LightOS cutover 中 `fe_sendauth: no password supplied`。ADR-027。
